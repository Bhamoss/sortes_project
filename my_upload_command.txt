/* 
avrdude is program to load
-v verbose output
-p MCU type
-c what will program your mcu = how will you put the binary on you hardware. Uses info from the config file . I think i can just say serial and remove -C
-C the config file for you programmer
-P the port to which you attach, the usb device (for serial port default is /dev/cuaa0)
-b the baud rate . arduino uses 57600 en internet dude 19200 met arduino als programmer en specifieert niets voor usbtiny
-U is used to perform a memory operation: -U memory_type:operation:filename[:format]
so -U flash:w:bin.hex  writes bin.hex to flash
format is to indicate which filetype = format you file is in, but can be autodetected. i is for indicating the file is a .hex file
-D disables auto erase. By default the flash of the device is erased before writing something new. 
ATxmega devices can use page erase so no explicit erase is required, so I guess this is an optimisation so you don't need to wait for a flash
*/
